# Open WebUI Pattern 1.1.0 Upgrade Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the Open WebUI Marketplace pattern from FOSS 0.6.36 (released 1.0.0) to FOSS 0.9.2, folding in the in-progress 1.0.1 secret-handling cleanup, and ship as 1.1.0 in time for the AWS-requested walkthrough video.

**Architecture:** Single AMI rebuild — bump `OPEN_WEBUI_VERSION` in the packer script, rebuild AMI, update `AMI_ID` constant + `NEXT_RELEASE_PREFIX` to `v110`, validate with `make synth`/deploy/integration tests/taskcat, then submit to AWS Marketplace per `aws-marketplace-utilities/UPGRADE.md` Phases 5–10. The product is in pre-launch evaluation with no live subscribers, so the DB migration in 0.9.0 affects only fresh deployments.

**Tech Stack:** Packer (Ubuntu 24.04 + CUDA 12.8.1 + cuDNN 9.8.0 + vLLM + Open WebUI), AWS CDK (Python, `aws-cdk-lib==2.225.0`, `oe-patterns-cdk-common@4.5.1`), CloudFormation, taskcat, pytest integration suite.

**Reference:** Follow [aws-marketplace-utilities/UPGRADE.md](file:///home/dylan/src/oe/patterns/aws-marketplace-utilities/UPGRADE.md) phases throughout; this plan only encodes the Open-WebUI-specific decisions and ordering.

## Pre-flight decisions

- **Target Open WebUI version:** 0.9.2 (latest stable at plan time; published 2026-04-24).
- **Target pattern version:** 1.1.0. Reasons: (a) FOSS bumps three minor versions (0.6 → 0.9), (b) Open WebUI 0.9.0 contains a database schema migration, (c) the existing `hotfix/1.0.1` branch already broke from the 1.0.0 line, so renaming is cheap.
- **Branch:** rename `hotfix/1.0.1` → `feature/upgrade` (per UPGRADE.md convention). Existing uncommitted work is kept and built on.
- **Laguna XS-2:** **defer.** Requires vLLM nightly (`pip install vllm --pre --extra-index-url https://wheels.vllm.ai/nightly`) until a vLLM stable release ships [vllm#41129](https://github.com/vllm-project/vllm/pull/41129). Pinning a nightly into a Marketplace AMI is not sustainable. Track in a follow-up issue; revisit when vLLM cuts a release with native support.
- **Devenv image:** confirm pinned to `ordinaryexperts/aws-marketplace-patterns-devenv:2.8.3` or newer.
- **AMI parameter suffix:** bump `AsgAmiIdv101` → `AsgAmiIdv110` (in CDK stack and Makefile).
- **DB migration risk:** Open WebUI 0.9.0 has a schema migration. Pattern uses singleton ASG with persistent `/data` EBS volume. Effect: on first boot of new AMI, Open WebUI will migrate the SQLite DB on the data volume in place. No action required from our side beyond explicit CHANGELOG mention. Pre-launch product → nobody to break.
